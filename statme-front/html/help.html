Игрокам

Этот сайт предназначен для удобного управления игровыми профилями, отслеживания прогресса и достижения ваших целей в играх. Мы предлагаем уникальную возможность не только для игроков, но и для компаний, разрабатывающих игры. Платформа объединяет этих пользователей в единую экосистему.

Возможности:

  • Создавать и редактировать профиль: В вашем профиле вы можете обновлять информацию о себе, добавлять или удалять игры, в которые играете, и отслеживать достижения.
  • Отслеживать прогресс: Мы предоставляем подробную статистику по вашим играм. Просто добавьте игру на сайт и войдите в аккаунт в игре. Компании сами будут отслеживать ваш прогресс.
  • Заработать достижения: Игры могут быть связаны с системой достижений, и мы отслеживаем, какие из них вы смогли получить. Это помогает не только следить за вашим успехом, но и стимулирует к дальнейшим достижениям.
  • Сохранять ваши любимые игры: На платформе можно добавлять игры в список интересующих. Вы сможете легко вернуться к ним в любое время и продолжить играть.


Компаниям:

Информация:

  • Разработка и публикация игр: Компании могут использовать платформу для создания игр и публикации их на сайте. Мы предоставляем все необходимые инструменты для того, чтобы отслеживание прогресса было удобным для Вас и Ваших игроков.
  • Отслеживание прогресса пользователей: Владелец компании может публиковать прогресс игроков по их достижениям в игре, что повышает интерес к игре.


API:

Все запросы (кроме авторизации) к API должны быть аутентифицированы при помощи вашего Bearer токена в заголовке "Authorization". Токен можно скопировать на странице игры.


1. Авторизация пользователя

POST, endpoint: /api/auth/login

Этот запрос позволяет войти в аккаунт пользователя.

Тело запроса:

{
    "email": "[почта]",
    "password": "[пароль]"
}

Ответ - токен авторизации пользователя.


2. Получение профиля пользователя

GET, endpoint: /api/profile/get

Этот запрос позволяет получить основную информацию о пользователе.

Если добавить в заголовок параметр id с идентификатором пользователя, можно получить информацию о другом пользователе.

Ответ:

{
    "id": Long,
    "email": String,
    "username": String,
    "photo": String or NULL,
    "status": String or NULL,
    "role": String
}

3. Получение списка достижений пользователя

GET, endpoint: /api/profile/achievements

Этот запрос позволяет получить список достижений пользователя.

Ответ:

[
    {
        "id": Long,
        "title": String
    },
    ...
]

4. Добавление достижения

POST, endpoint: /api/publish/achievement

Этот запрос позволяет компании добавить достижение пользователю, указав его ID и ID достижения.

Тело запроса:

{
    "userId": "[идентификатор пользователя]",
    "achievementId": "[идентификатор достижения]"
}
